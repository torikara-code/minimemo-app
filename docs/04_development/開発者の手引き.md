# Minimemo 開発者の手引き

開発中に発生する一般的な問題の解決方法や、開発フローについてのガイドです。

---

## 1. 開発サーバーの起動

以下のコマンドで、Vite の開発サーバーと Tauri アプリケーションを同時に起動します。

```bash
pnpm tauri dev
```

---

## 2. トラブルシューティング

### 「Port 1420 is already in use」エラーが発生する場合

Tauri のデフォルトポート `1420` が他のプロセス（前の開発セッションの残りなど）によって占有されている場合に発生します。以下の手順でプロセスを強制的に終了させることができます。

#### Windows (PowerShell)

1. **ポートを使用しているプロセスの PID（プロセス ID）を特定する**
   ```powershell
   netstat -ano | findstr :1420
   ```
   出力の右端にある数字（例: `15756`）が PID です。

2. **プロセスを kill する**
   ```powershell
   # 特定した PID を指定して終了
   taskkill /F /PID <PID>
   ```

3. **（オプション）画像名で一括 kill する**
   ```powershell
   # Node.js (Vite) を終了
   taskkill /F /IM node.exe /T
   # Tauri アプリ自体を終了
   taskkill /F /IM minimemo-app.exe /T
   ```

#### macOS (Terminal)

1. **ポートを使用しているプロセスを特定して kill する**
   ```bash
   lsof -i :1420
   # 表示された PID を指定して終了
   kill -9 <PID>
   ```

2. **ワンライナーで実行する場合**
   ```bash
   kill -9 $(lsof -ti:1420)
   ```

## 3. その他の便利なコマンド

*   **Rust 側の依存関係を更新する**: `cd src-tauri && cargo update`
*   **ビルド成果物をクリーンアップする**: `pnpm tauri build --debug -- --nocapture` (デバッグ用) または `rm -rf src-tauri/target`
