## 各ステップの詳細仕様

### [Step 1] 基盤構築とデザインシステム
*   **目的:** Minimemo の特徴である「LiquidGlass」デザインを定義し、レイアウトを確立する。
*   **タスク:**
    *   `index.css` / `App.css` に CSS 変数定義（ライト/ダークテーマ、透過、ブラー）。
    *   `App.tsx` の基本構造作成（メインコンテナ、エディタ枠、ステータスバー）。
    *   Google Fonts (Inter / Roboto Mono) の導入。
*   **完了条件:** デザイン仕様書通りの透過・フロストガラス効果がブラウザ上で確認できること。

### [Step 2] メインエディタと Markdown オーバーレイ
*   **目的:** 入力レスポンスを犠牲にしない「Markdown 簡易プレビュー」を実現する。
*   **タスク:**
    *   `TextArea` と `HighlightLayer` を重ねたエディタコンポーネントの作成。
    *   SolidJS の `createSignal` を用いた同期入力。
    *   正規表現による Markdown パーサーの実装（見出し、太字、リスト等）。
    *   カーソル位置やスクロールの完全同期。
*   **完了条件:** 入力したテキストに対してリアルタイムに Markdown 装飾がレイヤーとして重ねられること。

### [Step 3] Rust 側のウィンドウ制御とグローバルホットキー
*   **目的:** 常駐アプリとしての振る舞い（トグル表示、フォーカス制御）を実装する。
*   **タスク:**
    *   `src-tauri/src/lib.rs` での `Ctrl + M` グローバルホットキー登録。
    *   表示/非表示のトグルロジックと、表示時の強制フォーカス。
    *   システムトレイ（メニューバー）アイコンの追加。
*   **完了条件:** `Ctrl + M` を押すたびにウィンドウが現れたり消えたりし、現れた際にエディタにフォーカスが当たること。

### [Step 4] Data Persistence（保存機能）の実装
*   **目的:** メモの自動保存と永続化を実現する。
*   **タスク:**
    *   `tauri-plugin-store` のフロントエンド・ラッパー作成。
    *   タイピング停止後（デバウンス）およびウィンドウを閉じる際の保存処理。
    *   UUID v4 によるメモ ID 生成。
*   **完了条件:** アプリを再起動した際に、前回入力した内容が保持されていること。

### [Step 5] 履歴ドロップダウンとインタラクション
*   **目的:** 過去のメモをキーボードのみで呼び出せるようにする。
*   **タスク:**
    *   `Ctrl + K` による履歴パネルの展開/格納アニメーション。
    *   矢印キーによる履歴リストの選択移動。
    *   `Enter` での復元、`Delete` での削除（簡易アニメーション付き）。
*   **完了条件:** 履歴の一覧表示・選択・削除がキーボード操作のみで完結すること。

### [Step 6] 仕上げ：トースト通知とアニメーションの洗練
*   **目的:** 細部の UX を向上させ、プレミアムな体験を提供する。
*   **タスク:**
    *   `Ctrl + Enter` でのクリップボードコピー ＋ トースト表示。
    *   Expo-Out イージングを用いた、すべての UI 遷移の微調整。
    *   文字数・行数カウンターのリアルタイム更新。
*   **完了条件:** すべての操作に適切なフィードバックと滑らかなアニメーションが伴っていること。
