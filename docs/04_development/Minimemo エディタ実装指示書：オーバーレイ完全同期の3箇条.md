# ■ Minimemo エディタ実装指示書：オーバーレイ完全同期の3箇条

本ドキュメントは、SolidJSにおける `textarea`（入力層）と `div`（ハイライト表示層）の表示ズレを **0.1px単位で排除し、完璧なタイピング体験を実現する** ための技術仕様である。

---

## 第1条：幾何構造の絶対同一化（Visual Mirroring）
入力層と表示層のサイズ・余白・フォント計算に関わるすべてのCSSプロパティを、一字一句違わずに同一化せよ。

* **フォント特性:** `font-family`, `font-size`, `line-height`, `letter-spacing`, `font-variant-ligatures`（合字）を固定値で指定し、ブラウザの自動調整を無効化する。
* **ボックスモデル:** `padding`, `border-width: 0;`, `box-sizing: border-box;` を共通化し、両者を `position: absolute;` で親コンテナの左上 `(0, 0)` に固定する。
* **折り返しルール:** `white-space: pre-wrap;` および `word-wrap: break-word;` を両方に適用し、長文入力時の自動改行ポイントを物理的に一致させる。
* **スクロールバーの制御:** `textarea` の標準スクロールバーは表示幅を奪い、ズレの最大の原因となる。`scrollbar-width: none;` (Firefox) や `::-webkit-scrollbar { display: none; }` (Chrome/Edge) で非表示にするか、外側の共通コンテナでスクロールを管理すること。

---

## 第2条：スクロールベクトルの一時同期（Scroll Locking）
ユーザーが `textarea` を操作した際のスクロール位置（`scrollTop`, `scrollLeft`）を、表示層へ低レイテンシで転送せよ。

* **イベント駆動:** `textarea` の `onScroll` イベントをトリガーとする。
* **ダイレクト操作:** SolidJSのシグナル（State）を経由するとリアクティブ更新のオーバーヘッドが生じるため、`ref`（DOM要素への直接参照）を用いて、`highlightRef.scrollTop = textareaRef.scrollTop;` のように直接代入を実行すること。
* **双方向性の排除:** 入力層（textarea）をマスターとし、表示層（div）をスレーブとする一方向同期を徹底せよ。

---

## 第3条：末尾改行の不連続性解消（The Trailing Newline Hack）
HTMLの `div` 要素が「末尾の改行コード」を無視して要素高さを計算する仕様（ブラウザのレンダリング規則）を、プログミニメモ的に補正せよ。

* **ゴースト文字の挿入:** `textarea` 内のテキストが `\n`（改行）で終わっている場合、表示層へ渡すHTML文字列の末尾に必ず `&nbsp;` または半角スペースを1つ強制挿入する。
* **目的:** これを行わない場合、末尾でEnterキーを押した際にカーソル（textarea層）だけが次行へ移動し、ハイライト表示（div層）の高さが更新されず、描画位置が1行分ズレる現象が確定的に発生する。
