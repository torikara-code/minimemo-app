# Minimemo 実装方針：Acrylic LiquidGlass 仕様 (Tauri v2.2)

## 1. ウィンドウ基盤（Tauri + WinAPI）
Windows 11 のネイティブ・アクリル効果を最大限に引き出し、0.2秒の描画を実現する。

* **tauri.conf.json 設定:**
  - `transparent: true`: 背景透過の許可。
  - `decorations: false`: 標準枠（タイトルバー）を排除。
  - `shadow: false`: OS標準の影をオフ（自前で LiquidGlass 用の影を実装するため）。
  - `visible: false`: 起動時は非表示（バックグラウンド待機）。

* **Rust 側 (window-vibrancy + WinAPI):**
  - `apply_acrylic(window, None)` を実行。
  - 背景色（Alpha値）を `0` にし、OS 側のボカシエンジンがフロントエンドに貫通するように制御。
  - Windows 11 の角丸（DWMWA_WINDOW_CORNER_PREFERENCE）を Rust から強制適用し、アンチエイリアスの効いた美しい角を実現。

## 2. LiquidGlass 視覚スタック（SolidJS / CSS）
単なる透過ではなく、有機的な質感を出すための 4 層レイヤー構造。

1. **Base: OS Acrylic (Native)**
   - 背後のウィンドウをボカす。
2. **Layer 1: Tint Filter (CSS)**
   - `background: rgba(255, 255, 255, 0.45);`
   - ガラスの色味を決定。ライト/ダークモードに応じて透過率を微調整。
3. **Layer 2: Grain Texture (CSS)**
   - 100x100px 程度の微細なノイズ画像を `repeat` で配置。
   - `opacity: 0.03; mix-blend-mode: overlay;`
   - デジタル特有の「安っぽさ」を消し、実在する物質感を付与。
4. **Layer 3: Edge & Shadow (CSS)**
   - `border: 1px solid rgba(255, 255, 255, 0.4);`
   - `border-top: 1.5px solid rgba(255, 255, 255, 0.7);`（上部光源のシミュレート）
   - `box-shadow: 0 24px 48px rgba(0, 0, 0, 0.16);`（浮遊感の演出）

## 4. ゼロ・ラグ・タイピング（パフォーマンス）
* **CSS Blur の禁止:**
  - `backdrop-filter: blur()` は計算負荷が極めて高いため、絶対に使用しない。ボカシは OS（Acrylic）に完全に任せる。
* **低遅延レンダリング:**
  - SolidJS の `createSignal` を使い、テキスト入力時はエディタ部分のみを再描画。背景レイヤーは静止状態を維持。

## 5. ライフサイクル管理
* **Show/Hide トグル:**
  - `Ctrl + M` 押下時、`window.show()` 直後に `window.focus()` を実行。
  - `Esc` 押下時、即座に `window.hide()`。`close` しないことで再起動コストをゼロにする。
* **履歴アニメーション:**
  - 履歴展開時は `transition: height 0.25s cubic-bezier(0.16, 1, 0.3, 1)` を使用し、ガラスが下に伸びるような滑らかな挙動を実装。
