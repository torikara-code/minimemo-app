# ■ Minimemo システムアーキテクチャ・内部設計書

## 1. システム構成 (Tauri v2.2 + SolidJS)
MinimemoはTauriフレームワークを利用し、以下の2層アーキテクチャで構成する。

* **フロントエンド (Webview):** SolidJS v1.9 / TypeScript 5.6
    * 役割: UIの描画、シグナルによる状態管理、Markdownハイライト、アニメーション制御。仮想DOMを使わないため、Vanilla JSに匹敵する速度で動作。
* **バックエンド (Rust Core):** Rust 1.78+
    * 役割: `tauri-plugin-store` v2 によるデータ永続化、グローバルショートカット（`Ctrl + M`）の監視、ウィンドウの表示/非表示（透過・最前面）制御。

---

## 2. データ永続化設計 (tauri-plugin-store v2)
データの永続化には SQLite の代わりに、Tauri公式プラグインの `tauri-plugin-store` v2 を使用し、JSON ファイルで管理する。

### 保存パス: `~/.config/Minimemo/data.json`

### データ構造
```typescript
interface AppData {
  memos: Memo[];
  settings: Settings;
}

interface Memo {
  id: string; // UUID v4
  content: string;
  updated_at: number; // UNIXタイムスタンプ
}

interface Settings {
  theme: 'light' | 'dark';
  history_limit: number;
}
```

## 3. IPC (インタープロセス通信) API仕様
SolidJS から Tauri バックエンドを呼び出すコマンド、およびプラグインの利用。

### 3.1 ウィンドウ制御 (Rust Custom Commands)
* **`hide_window`**
    * 概要: ウィンドウを隠す（バックグラウンド待機）。
* **`copy_to_clipboard`**
    * 概要: 指定したテキストをOSのクリップボードにコピーする。

### 3.2 データ管理 (tauri-plugin-store)
フロントエンドの SolidJS コンポーネントから直接、またはラップした API 関数経由でプラグインを利用する。
* **`store.set(key, value)`**: データの書き込み。
* **`store.get(key)`**: データの読み込み。
* **`store.save()`**: ファイルへの永続化。

## 4. ウィンドウライフサイクルと状態管理

### 4.1 初期化（起動時）
1.  Rust側でTauriアプリをビルド。
2.  ウィンドウ設定: `transparent: true`, `decorations: false`, `always_on_top: true`, `visible: false`。
3.  フロントエンド起動時に `tauri-plugin-store` から最新のデータをロードし、SolidJS の Signal（Store）を初期化。

### 4.2 呼び出し（`Ctrl + M`）
1.  Rust側で登録したグローバルショートカットが発火し、ウィンドウを表示（`window.show()`）。
2.  フロントエンドへフォーカス要求イベントを送信。
3.  SolidJS 側でイベントを検知し、エディタ要素にフォーカスを当てる。

### 4.3 終了・格納
1.  ユーザーが `Esc` や `Ctrl + Enter` を入力。
2.  SolidJS 側で直前の入力を Store に保存。
3.  `tauri-plugin-store` の `save()` を実行。
4.  Rust 側の `hide_window` を呼び出し、ウィンドウを隠す。
