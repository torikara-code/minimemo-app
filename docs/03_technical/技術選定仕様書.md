
# ■ Minimemo 全体的技術選定・アーキテクチャ仕様書（爆速・SolidJS版 / 2026アップデート）


## 1. 全体システム構成
Minimemoは「究極の軽さ」と「ミリ秒のレスポンス」を達成するため、バックエンドにRust（Tauri v2系）、フロントエンドに仮想DOMを持たないSolidJSを採用した超軽量アーキテクチャとする。


| レイヤー | 採用技術 | 選定理由・目的 |
| :--- | :--- | :--- |
| **コア/バックエンド** | **Tauri v2.2 (Rust 1.78+)** | Electron比で圧倒的軽量。v2でプラグイン統合・トレイ安定化・非同期処理が強化。ネイティブAPI制御も高速。 |
| **フロントエンドUI** | **SolidJS v1.9 + SolidStart (SPAモード)** | ファイングレインリアクティビティにより仮想DOM不要。v1.9でパフォーマンスと開発体験が向上。 |
| **開発言語** | **TypeScript 5.6 (ES2023)** | 型安全性 + 最新JS最適化。大規模化時の保守性向上。 |
| **スタイリング** | **CSS Modules または Vanilla Extract** | ランタイムゼロを維持しつつ型安全も確保可能。UIが単純なため軽量維持。 |
| **ローカルDB** | **tauri-plugin-store v2 (JSON)** | 軽量・高速・スレッドセーフ。小規模データではSQLiteより低オーバーヘッド。 |
| **ビルド環境** | **Vite 6 + vite-plugin-solid** | 高速HMR・キャッシュ最適化・Tauriとの相性良好。 |
| **依存管理** | **pnpm v9** | インストール速度・ディスク効率が高い。 |

## 2. フロントエンド設計（SolidJSアーキテクチャ）


SolidJSの `createSignal` / `createEffect` を活用し、EventBusを廃止。必要最小限のグローバル状態のみを共有し、他はコンポーネントローカルに閉じる。


### 2.1 ディレクトリ構成
```text
src/
 ├── index.html
 ├── index.tsx
 ├── App.tsx
 ├── styles/
 │    └── global.css
 ├── store/
 │    └── appState.ts
 ├── components/
 │    ├── Editor.tsx
 │    ├── History.tsx
 │    └── StatusBar.tsx
 └── api/
      └── tauri.ts
```


### 2.2 状態管理（store/appState.ts）

グローバルState（最小限）：
- [text, setText]
- [isHistoryOpen, setIsHistoryOpen]
- [historyItems, setHistoryItems]

ローカルState（例）：
- toastMsg（StatusBar内で管理）

### データフロー（文字入力）
1. Editorで入力
2. setText実行
3. 依存DOMのみ即時更新（仮想DOMなし）

### 3.3 トレイ連携
```rust
fn toggle_window(app: tauri::AppHandle) {
  let window = app.get_window("main").unwrap();
  if window.is_visible().unwrap() {
    window.hide().unwrap();
  } else {
    window.show().unwrap();
  }
}
```

## 4. 爆速UX実装ルール


1. タイピング遅延排除  
   Markdownパースは軽量維持（正規表現最適化）

2. 常駐プロセス化  
   window.hide()のみ使用し、プロセスは維持

3. GPUアクセラレーション  
   transform / opacity のみ使用  
   + will-change指定で初動遅延ゼロ化

4. State最小化  
   グローバルSignalを必要最低限に制限

5. invoke最適化  
   Rust側完全非同期でUIブロック防止
