# ■ Minimemo エラーハンドリング・ログ管理仕様書

## 1. 基本方針（Error Handling Philosophy）
「思考の外部バッファ」という性質上、エラー通知によってユーザーの作業をブロックする（画面中央に巨大な警告ダイアログを出すなど）ことは極力避ける。
状態に応じて「トースト通知」「ステータスバーでの静かな警告」「バックグラウンドログ」を使い分け、堅牢かつノイズレスなUXを担保する。

---

## 2. エラーレベルと通知UIの定義

### レベル1: 致命的エラー (Fatal)
**定義:** アプリの継続利用が不可能、またはデータが消失する危険がある状態。
**通知方法:** ステータスバー（画面下部）を赤色（`--danger`）に変更し、常時警告を表示。

* **シナリオ A: データベース（tauri-plugin-store / SQLite）の初期化失敗**
    * **原因:** 権限不足、ディスク容量不足、ファイル破損など。
    * **バックエンド(Rust):** パニック（クラッシュ）させず、インメモリ（一時保存）モードへフォールバックし、フロントエンドに `DB_INIT_FAILED` イベントを送信。同時にエラーログをファイルに書き出す。
    * **フロントエンド(UI):** ステータスバーを「⚠️ DB Error: Read-only mode」とし、設定画面に詳細エラーを表示する。

* **シナリオ B: 自動保存の失敗**
    * **原因:** I/Oエラーなど。
    * **バックエンド(Rust):** 3回リトライし、それでもダメならエラーイベントを送信。
    * **フロントエンド(UI):** トースト通知で「⚠️ Failed to save. Please copy your text.（保存失敗。テキストをコピーしてください）」と表示（赤色トースト）。

### レベル2: 警告 (Warning)
**定義:** アプリの起動・入力はできるが、一部の機能（特にUXのコア部分）が阻害されている状態。
**通知方法:** 起動時に1度だけトースト通知を表示 ＋ 設定画面（Settings）にアラートバッジを表示。

* **シナリオ C: グローバルショートカット（`Ctrl + M`）の競合**
    * **原因:** 他の常駐アプリ（Discord, Notion, OS標準機能など）が既に同じキーボードフックを取得している。
    * **バックエンド(Rust):** `register` 失敗時にクラッシュさせず、システムトレイ（タスクトレイ）アイコンのみで起動を継続。フロントエンドに `SHORTCUT_CONFLICT` イベントを送信。
    * **フロントエンド(UI):** * 起動時にトースト表示: 「Shortcut (Ctrl+M) is used by another app.」
        * 設定画面の歯車アイコンに赤ポチ（バッジ）を付け、設定画面内で「別のショートカットキーを割り当ててください」と促すUIを表示。

### レベル3: サイレントエラー (Silent)
**定義:** ユーザー体験に直結しない、内部的な一時エラー。
**通知方法:** UIには一切出さず、バックエンドのログファイルにのみ記録する。

* **シナリオ:** Markdownのパース遅延、OSネイティブ通知の送信失敗など。

## 3. ロギング基盤（Backend / Rust）

エラー調査やユーザーからのバグレポートに対応するため、Rust側でファイルロギングを実装する。

* **使用クレート:** `log` および `log4rs`（またはTauri公式の `tauri-plugin-log`）。
* **ログ出力先:** OS標準のアプリケーションデータディレクトリ。
    * Windows: `C:\Users\{User}\AppData\Roaming\com.minimemo.app\logs\app.log`
    * macOS: `/Users/{User}/Library/Logs/com.minimemo.app/app.log`
* **ログローテーション:** ログファイルが 5MB を超えたら古いものをアーカイブ（最大3世代まで保持）。ディスクを圧迫しない設計。

## 4. フロントエンドからRust関数を呼ぶ際のエラー捕捉（SolidJS）

フロントエンド（API層）で `invoke` を実行する際は、必ず `try...catch` で囲み、エラーを握りつぶさないように実装する。

```typescript
// src/api/tauri.ts の実装例
import { invoke } from '@tauri-apps/api/tauri';
import { showToast } from '../components/Toast';

export async function saveMemoToDB(content: string) {
  try {
    await invoke('save_memo', { content });
  } catch (error) {
    console.error("Failed to save memo:", error);
    // イベントバス（またはSignal）経由でエラートーストを表示
    showToast('Failed to save', 'error');
  }
}
```
