# バックエンド開発標準（Rust / Tauri v2.2）

Minimemoのバックエンドは、OSネイティブ機能の制御、ウィンドウ管理、およびデータの安全な永続化を担当します。

## 1. 役割の定義
- **ウィンドウ制限制御**: 透過、枠なし、常に最前面、表示/非表示の切り替え。
- **データ永続化**: `tauri-plugin-store` v2 を用いた JSON ベースの保存。
- **OS連携**: グローバルショートカット (`Ctrl + M`)、クリップボード操作、システムトレイ。
- **ロギング**: アプリケーションログの出力とローテーション。

## 2. ウィンドウ管理の原則
Minimemoは「常駐型」アプリであるため、以下のルールを厳守してください。
- **非表示起動**: 起動時は `visible: false` で開始し、トレイまたはショートカットで表示する。
- **破棄禁止**: メインウィンドウを閉じる操作（Esc等）は `window.close()` ではなく `window.hide()` を使用し、プロセスを維持すること。
- **透過と最前面**: `tauri.conf.json` で `transparent: true`, `alwaysOnTop: true`, `decorations: false` を設定。

## 3. IPC (Internal Process Communication)
フロントエンドとの通信には `tauri::command` を使用します。
- **非同期処理**: I/Oを伴う処理は必ず `async` 関数として定義する。
- **エラー返却**: `Result<T, E>` 型を使用し、フロントエンドが適切にエラーハンドリング（トレイ通知等）できるようにする。
- **データ構造**: 保存・通信に使用する構造体には `serde::Serialize` / `Deserialize` を派生させる。

## 4. データ永続化 (`tauri-plugin-store` v2)
SQLite を使わず、公式プラグインで JSON 保存を行います。
- **保存パス**: `~/.config/Minimemo/data.json`
- **整合性**: 重大な更新の後は `store.save()` を明示的に呼び出し、ファイルへの書き込みを確定させる。

## 5. エラーハンドリングとロギング
- **パニック回避**: `unwrap()` の多用を避け、エラーを伝播させる。
- **ログレベル**: 
    - `Error`: 起動失敗、DB破損、ショートカット登録失敗。
    - `Warn`: 権限不足、リトライが必要なI/O。
    - `Info`: ウィンドウの表示/非表示の状態変化、設定保存。
- **ログ出力先**: OS標準のアプリケーションデータディレクトリ（詳細は仕様書参照）。

## 6. ショートカットとトレイ
- **ショートカット競合**: `register` 失敗時はトレイアイコンのみで起動を継続し、フロントエンドへ `SHORTCUT_CONFLICT` イベントを通知する。
- **トレイ制御**: 右クリックメニューで「Quit」を用意し、完全にプロセスを終了させる手段をユーザーに提供する。

## 7. AI 動作指針
1. `src-tauri` 内のコードを変更する際は、Tauri v1 ではなく v2 のAPI体系（`tauri::AppHandle` 等）を使用せよ。
2. 新しいコマンドを追加した場合は、`main.rs` の `generate_handler!` への登録を忘れないこと。
3. セキュリティのため、フロントエンドから渡される文字列（特にファイルパスやコマンド引数）は、実行前に必ずバリデーションせよ。
