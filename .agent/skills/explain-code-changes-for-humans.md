[目的]
このスキルは、コードの変更や追加が行われた際に、そのファイルの実装意図を「1行たりとも逃さず」技術的に詳細解説するためのルールです。
目的は、その言語の初学者であっても、ドキュメントを読むだけで「依存関係のロードから、各行の処理の意図」までを完全に把握できるようにすることです。
擬人化や過剰な比喩（魔法、大臣、守護神など）は一切禁止し、純粋な技術的メカニズムに基づいた解説を行います。

----------------------------------------------------------------------
【適用タイミング】
----------------------------------------------------------------------
以下の操作が行われた直後に、対応する解説ファイルを生成または最新状態へ更新してください。

- 新規ファイルの作成
- 既存ロジックの変更（たとえ 1 行であっても）
- 依存関係（use / import）の追加・変更

----------------------------------------------------------------------
【出力ファイルパスのルール】
----------------------------------------------------------------------
ソースコードのディレクトリ構造を `docs/changes/` 配下にそのままミラーリングします。

- 形式: `docs/changes/[元のファイルパス].md`
- 例: 
  - `backend/src/lib.rs` → `docs/changes/backend/src/lib.rs.md`
  - `frontend/src/utils/api.ts` → `docs/changes/frontend/src/utils/api.ts.md`

----------------------------------------------------------------------
【ドキュメント構成（極限・全行解説モード）】
----------------------------------------------------------------------

1. 概要
- ファイルの主目的を 3 行以内で技術的に記述。比喩表現は厳禁。

2. 依存関係の詳解（1 行も逃さない）
- **`use` や `import` されている全ての行** について、そのライブラリやモジュールが「なぜ必要なのか」「このファイルでどの型や関数を提供しているか」を 1 つずつ解説してください。

3. 型・構造体定義の詳細
- 定義されている全ての型（struct, enum, interface等）について、各フィールドのデータ型、保持する情報の意味、用途を網羅。

4. 実装ロジックの精査（1 行・1 ブロックも漏らさない）
- **全ての関数・メソッド** について、以下の構成で記述：
  - **関数名とインターフェース**: 引数の役割、戻り値の型と意味（成功/失敗のケース）。
  - **内部処理のステップ解説（極めて詳細に）**:
    - コードの全行を論理的な最小単位で分割し、「変数の初期化」「条件分岐の判定基準」「エラー投げる際の意図」など、**全ての行の意図** を記述。
    - 複雑な箇所や言語特有の動作（Rustの所有権、ライフタイム等）は、詳細に補足。

5. 技術要素の解説
- 使用されている言語機能（`?`, `match`, `async/await`, `traits` 等）を、動作原理に基づいて簡潔に解説。

6. データフローの追跡
- 入力値から出力値に至るまでのデータの変化を時系列で追跡。

----------------------------------------------------------------------
【厳守事項】
----------------------------------------------------------------------
1. **「比喩の完全排除」**:
   「魔法」「大臣」などの表現は厳禁。正確な技術用語を使用すること。

2. **「1 行（依存関係含む）も逃さない」**:
   `use` 句からロジックの末尾まで、コードに存在する全ての論理的なステップに、開発者の意図としての解説を付与すること。

3. **「最新性の維持」**:
   コードとドキュメントを常に同期させ、常に「現在のファイル全体の状態」を解説した最新版を維持すること。
